# Installing Alpine Linux on Sophos XG 115 via Serial Console

## Prerequisites
- USB-to-serial cable (Cisco console cable works)
- Terminal emulator configured for 115200 baud, 8N1, no flow control
- Rufus (Windows) for creating the bootable USB
- Alpine Linux ISO (latest)

## Step 1 — Create Bootable USB
Use Rufus in **ISO Image mode** (not DD mode) to write the Alpine ISO. This creates an editable FAT32 filesystem on the USB.

## Step 2 — Configure USB Syslinux for Serial Console
Navigate to the USB drive and edit two files:

**File 1: `\syslinux.cfg`** (root of USB)

Verify it contains the loadconfig stub — no changes needed here beyond confirming it exists.

**File 2: `\boot\syslinux\syslinux.cfg`**

Add `SERIAL 0 115200` as the absolute first line, add `console=ttyS0,115200n8` to the APPEND line, and remove `quiet`:

```properties
SERIAL 0 115200
TIMEOUT 10
PROMPT 1
DEFAULT lts

LABEL lts
MENU LABEL Linux lts
KERNEL /boot/vmlinuz-lts
INITRD /boot/initramfs-lts
FDTDIR /boot/dtbs-lts
APPEND modules=loop,squashfs,sd-mod,usb-storage console=ttyS0,115200n8
```

> Note: SYSLINUX may not accept keyboard input at the `boot:` prompt despite the SERIAL directive. The 1-second TIMEOUT autoboot handles this — just wait and it will proceed automatically.

## Step 3 — BIOS Configuration
Boot the Sophos XG 115 and enter BIOS via serial console. Ensure the USB drive is first in the boot order.

## Step 4 — Install Alpine
Once booted into Alpine from USB, run the installer:

```sh
setup-alpine
```

Key choices during install:
- Select your target disk (typically `sda`) — confirm it is **not** your USB stick
- Choose `sys` install mode for a full disk installation

## Step 5 — Configure Installed Bootloader Before Rebooting
**Do this before rebooting.** The installed bootloader has no serial console config by default and the system will go silent on first boot if you skip this.

Mount the installed system's boot partition:

```sh
mount /dev/sda1 /mnt
```

Edit `/mnt/boot/extlinux.conf` to add `SERIAL 0 115200` as the first line and append `console=ttyS0,115200n8` to the APPEND line. Remove `quiet`. The final file should look like:

```properties
SERIAL 0 115200
# Generated by update-extlinux
DEFAULT menu.c32
PROMPT 0
MENU TITLE Alpine/Linux Boot Menu
MENU HIDDEN
MENU AUTOBOOT Alpine will be booted automatically in # seconds.
TIMEOUT 10
LABEL lts
  MENU DEFAULT
  MENU LABEL Linux lts
  LINUX vmlinuz-lts
  INITRD initramfs-lts
  APPEND root=UUID=<your-uuid> modules=sd-mod,usb-storage,ext4 rootfs console=ttyS0,115200n8
MENU SEPARATOR
```

Unmount cleanly before rebooting:

```sh
umount /mnt
```

## Step 6 — Reboot to Installed System
Remove the USB stick and reboot. You should see full boot output on the serial console and land at an Alpine login prompt.

---

**Key Principle:** Every bootloader in the chain requires the serial console configuration explicitly. Nothing inherits it automatically from BIOS or from a previous bootloader stage.
